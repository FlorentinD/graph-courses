= Neo4j Data ImporterでCSVファイルをインポートする
:type: challenge
:sandbox: true
:updated-at: 2022-05-02 T 13:00:00 Z

このChallengeでは、前モジュールで見たムービーデータをインポートします。

このチャレンジは以下の12のステップで構成されています:

. CSVファイルをダウンロードする
. データインポーターを開く
. CSVファイルをData Importerに読み込む
. Person ノードのマッピングを定義する
. Movieノードのマッピングを定義する
. Userノードのマッピングを定義する
. DIRECTED リレーションシップのマッピングを定義する
. ACTED_INリレーションシップのマッピングを定義する
. RATEDリレーションシップを定義する
. マッピングを保存する
. CSVデータをインポートする
. インポートしたデータを確認する

今回実装するグラフデータモデルは以下の通りです:

image::images/movie-data-model.png[Movie Data Model,width=600,align=center]

プロパティの型に注意インポート用のマッピングを定義する際、自動的にマッピングできる型（文字列、整数、デシマル）に注意する必要があり、プロパティのキー値はCSVファイルにあるものと異なる可能性があります。

== Step 1: CSVファイルをダウンロードする

以下の https://data.neo4j.com/importing/movieData.zip[file^] をお使いのシステムにダウンロードしてください。

このファイルを解凍してください。以下のファイルがあるはずです:

* persons.csv
* movies.csv
* ratings.csv
* directed.csv
* acted_in.csv

== Step 2: データインポーターを開く

Web ブラウザで、 https://data-importer.graphapp.io/?acceptTerms=true[Neo4j Data Importer^] を開きます。

Data Importerを開いて、このコースのサンドボックスインスタンスに接続する必要があります。
データをインポートする前に、インポート先の Neo4j インスタンスの情報を入手しておく必要があります。
ホストロケーションは、このコースのために作成されたサンドボックスの WebSocket ボルト URL である必要があります。
サンドボックスは、デフォルトでは3日間だけ存在し、コースを継続するときに再作成されます。
もし、メールで送られた認証情報が使えない、または持っていない場合は、以下のようになります:

. https://sandbox.neo4j.com[sandbox site^] にアクセスしてください。
. ログインして、空白のサンドボックス・インスタンスの詳細を確認します。
. サンドボックスの右端のドロップダウンをクリックすると、空のサンドボックスの詳細が表示されます。ここで、接続の詳細が表示されます。

image::images/blank-sandbox-credentials.png[Blank Sandbox credentials,width=600,align=center]

このダイアログにプロトコルとしてbolt+s、WebSocketのURL、パスワードを入力し、**Connect** をクリックします。

image::images/connect.png[Connect to DBMS,width=600,align=center]

このように表示されるはずです:

image::images/data-importer.png[Neo4j Data Importer,width=600,align=center]

[NOTE]
--
Neo4j Data Importerは、このコースが公開された後に変更される可能性があります。
UIに細かな違いが見られるでしょう。

グラフモデルペインにノードやリレーションシップが表示されている場合は、それらをすべて選択して削除し、新しいマッピングで開始できるようにします。
--
== Step 3: CSVファイルをData Importerに読み込む

左の *Files* パネルに、*Step1* で解凍した5つのCSVファイルを追加します。

追加した後、このように表示されるはずです:

image::images/files-selected.png[Files added to Data Importer,width=600,align=center]

読み込んだら、各ファイルに使用されているヘッダーを数分かけて確認してください。

== Step 4: Person ノードのマッピングを定義する

. グラフモデルペインのノード追加アイコンをクリック
. 右側のマッピングの詳細ペインで:
.. ラベルには *Person* と入力
.. *persons.csv* ファイルを選択
.. プロパティで**Add from File**をクリック
... *All* を選択
... *Confirm* をクリック
.. いくつかのプロパティ名がグラフのデータモデルと一致していないことに注意してください。それらを以下のように修正:
... person_tmdbId --> tmdbId
... person_imdbId --> imdbId
... person_poster --> poster
... person_url --> url
.. Personノードに使用される一意のIDとして、*tmdbId* を選択

左側のパネルに、すべてのプロパティがマッピングされたことを示す緑色のインジケータが表示されます:

image::images/person-properties-mapped.png[Person properties mapped,width=400,align=center]

== Step 5: Movieノードのマッピングを定義する

. グラフモデルペインのノード追加アイコンをクリック
. 右側のマッピングの詳細ペインで:
.. ラベルに *Movie* と入力
.. *movies.csv* ファイルを選択
.. プロパティでファイルから追加をクリック
... *All* を選択
... *Confirm* をクリック
.. いくつかのプロパティ名がグラフのデータモデルと一致していないことに注意してください。それらを以下のように修正:
... movie_tmdbId --> tmdbId
... movie_imdbId --> imdbId
... movie_poster --> poster
... movie_url --> url
.. また、いくつかのプロパティタイプは、データモデルが定義するものと一致しないことに注意してください。以下のようにプロパティの種類を変更します:
... budget --> integer
... imdbRating --> float
... imdbVotes --> integer
... revenue --> integer
... runtime --> integer
... year --> integer
.. ムービーノードに使用されるユニークIDとして *movieId* を選択します。movieIDをユニークIDとして選択したのは、他のファイルでムービーとのリレーションシップを定義する際に使用されるからです。

[NOTE]
データインポーターでは、すべてのフィールドを目的のプロパティタイプに変換することはできません。このコースの次のモジュールでは、グラフのインポート後のリファクタリングを実行します。

左側のパネルに、すべてのプロパティがマッピングされたことを示す緑色のインジケータが表示されるはずです:

image::images/movie-nodes-mapped.png[Movie nodes mapped,width=600,align=center]

== Step 6: Userノードのマッピングを定義する

. グラフモデル]ペインの[ノード追加]アイコンをクリックします。
. 右側の「マッピングの詳細」ペインで:
.. ラベルには *User* と入力します。
.. *ratings.csv* ファイルを選択します。
.. Under Properties, click *Add from File*.
... Select only the *userId* and *name* properties. The other values will be used later to define relationships.
... Click *Confirm*.
.. Select *userId* as the unique ID that will be used for these User nodes.

You should see a green indicator in the left panel that all the properties have been mapped:

image::images/user-nodes-mapped.png[User nodes mapped,width=600,align=center]

== Step 7: DIRECTED リレーションシップのマッピングを定義する
// Elaine - resume here
. In the Graph Model pane, drag the edge of the *Person* node onto the *Movie* node. This will define a relationship between these nodes in the graph.
. In the Mapping Details pane on the right:
.. Enter *DIRECTED* for the type.
.. Select the *directed.csv* file.
.. In the IDs section:
... Select person_tmdbId as the Person ID.
... Select movieId as the Movie ID.

There are no fields in the *directed.csv* file for role so you will not be importing the role property from this file, so you are done.

You should see a green indicator in the left panel that all the properties have been mapped:

image::images/confirm-DIRECTED-done.png[DIRECTED relationships mapped,width=600,align=center]

== Step 8: ACTED_INリレーションシップのマッピングを定義する

. In the Graph Model pane, drag the edge of the *Person* node onto the *Movie* node. This will define a relationship between these nodes in the graph.
. In the Mapping Details pane on the right:
.. Enter *ACTED_IN* for the type.
.. Select the *acted_in.csv* file.
.. In the IDs section:
... Select person_tmdbId as the Person ID.
... Select movieId as the Movie ID.
.. In the Properties section, select the *role* property.

You should see a green indicator in the left panel that all the properties have been mapped:

image::images/acted-in-relationship-mapped.png[ACTED_IN relationships mapped,width=600,align=center]

== Step 9: RATEDリレーションシップを定義する

このマッピングは、Userノードを定義するために使用したのと同じファイルを再利用することに注意してください。

. グラフモデルペインで、*User* ノードのエッジを *Movie* ノードの上にドラッグします。これで、グラフ内のこれらのノード間のリレーションシップが定義されます。
. 右側のマッピングの詳細ペインで:
.. タイプは *RATED* を入力してください。
.. *ratings.csv* ファイルを選択します。
.. IDsセクションで:
... ユーザーIDとしてuserIdを選択します。
... ムービーIDとしてmovieIdを選択します。
.. Propertiesセクションで、*rating* と *timestamp* のプロパティを選択します。
.. いくつかのプロパティタイプは、グラフのデータモデルと一致しないことに注意してください。それらを以下のように修正する。
... rating --> integer
... timestamp --> integer

左側のパネルに、すべてのプロパティがマッピングされたことを示す緑色のインジケータが表示されるはずです:

image::images/rated-relationships-mapped.png[RATED relationships mapped,width=600,align=center]

== Step 10: RATEDリレーションシップの定義

インポート中に何か問題が発生した場合、マッピングしたものを保存しておくことをお勧めします。

. *Run Import* ボタンの右側にある *...* をクリックします。
. *Download Model* を選択します。*neo4j_importer_model_yyyy-mm-dd.json* という名前のファイルが、あなたのシステムにダウンロードされます。

image::images/download-model.png[Save the model,width=600,align=center]

== Step 11: CSVデータをインポートする

以下の手順でデータを取り込みます:

. *Run Import* ボタンをクリックします。
. 空のグラフにデータがインポートされるはずです。(2、3分かかります)。

image::images/import-completed.png[Import completed,width=600,align=center]
